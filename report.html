1. Problem description

This website is a collection of brief history on Linux and various Linux distributions such as Ubuntu, Arch linux, Fedora linux and Linux mint. It also offers brief description of above mentioned distributions as well as system requirements, compatible desktop enviroments and currently supported versions of such distributions.

Users of this site might be people who are looking to move away from Windows or MacOS to Linux and are in search of worthwile replacement. They might also just look for brief a brief video of listed Linux distribution to educate themselves on the layout of given distributions. Lastly, some of the previous Linux users might just want to catch up on the current state of the given distribution.

The site solves the problem of scattered and misclassified data and media about above mentioned distributions.

2. Research

For the research, I looked at few web sites which are offering some of the above mentioned distributions. All of them are offering plenty of material to work with, but it might seem as "too much information" for a regular user looking to move their personal computer to linux based OS. When I was researching various linux distributions, it felt like there is too much "power user" information and not enough information for a regular user to ease into Linux OS and their respective desktop enviroments.

3. Technology Selection and Site Architecture

For design and implementation of this web site, I used HTML5, CSS 2.1 as well as Javascript.

HTML5

TML5 is a markup language used for structuring and presenting content on the World Wide Web. It is the fifth and last major HTML version that is a World Wide Web Consortium (W3C) recommendation. The current specification is known as the HTML Living Standard. It is maintained by the Web Hypertext Application Technology Working Group (WHATWG), a consortium of the major browser vendors (Apple, Google, Mozilla, and Microsoft).

CSS

Cascading Style Sheets (CSS) is a style sheet language used for describing the presentation of a document written in a markup language such as HTML. CSS is a cornerstone technology of the World Wide Web, alongside HTML and JavaScript. CSS is designed to enable the separation of presentation and content, including layout, colors, and fonts. This separation can improve content accessibility; provide more flexibility and control in the specification of presentation characteristics; enable multiple web pages to share formatting by specifying the relevant CSS in a separate .css file, which reduces complexity and repetition in the structural content; and enable the .css file to be cached to improve the page load speed between the pages that share the file and its formatting.

Javascript

JavaScript, often abbreviated as JS, is a programming language that conforms to the ECMAScript specification. JavaScript is high-level, often just-in-time compiled and multi-paradigm. It has dynamic typing, prototype-based object-orientation and first-class functions. Alongside HTML and CSS, JavaScript is one of the core technologies of the World Wide Web. Over 97% of websites use it client-side for web page behavior, often incorporating third-party libraries. All major web browsers have a dedicated JavaScript engine to execute the code on the user's device. As a multi-paradigm language, JavaScript supports event-driven, functional, and imperative programming styles. It has application programming interfaces (APIs) for working with text, dates, regular expressions, standard data structures, and the Document Object Model (DOM).

5. Development Plan

1. Research

I hope for this website to educate the audience on above mentioned linux operating system distributions with giving a brief overview on each Linux flavour.
The audience I am trying to reach are users with little or no previous experience with unix-like systems and previous linux users looking to catch up with current state of Linux systems.
The goals of the website are to present the topic content in clear and meaningful way with usage of HTML, CSS and Javascript as well as interest users into trying or switching their operating system to Linux.

2. Design and planning

This step involved designing the wireframe and sitemap of the website. For that I used a simple photo editor. Planning was very important step since good site layout is crucial to ensure user will easily navigate the site. In this step I have also designed gradients for background, navigation bar and footer.

3. Writing the content

In this step the actual content for the website was written and formatted.

4. Coding the website

This part took the longest since there was a lot of components on the website that had to look good and still deliver the content in a meaningful way. The website had to look appealing, but not too flashy to distract the reader from the content itself.

5. Testing

For this phase various website test tools will be used as well as wide range of browsers and different platforms to confirm that website displays everywhere as intended.

6. Testing Plan

For testing I have used the W3C Markup Validation Service to validate the HTML and W3C CSS Validation Service to validate CSS. Website display testing was done through various different browsers such as Firefox, Chrome, Chromium, Brave, Microsof Edge, Opera, Safari, and yes, Internet explorer. The website was self-hosted on a home Linux server machine to allow for easy access to the website from varous different machines/platforms.

7. Site Evaluation

The site was evaluated with online tools to verify there are no issues with accessibility of content and to conform with HTML5 standards. The site is fully usable with text-to-speech tool as well as fully responsive, meaning it will fit perfectly to any size of the display/device.

I have used free online all in one tool called accessibiliychecker.org
